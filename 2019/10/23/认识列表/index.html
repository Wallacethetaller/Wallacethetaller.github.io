认识列表

Posted on 2019-10-23

#1. 什么是列表(list)
##1. 列表
列表是容器型数据类型（序列），将’[]’作为容器的标志，每个元素用逗号隔开，形如[元素1, 元素2, 元素3, …]。
列表作为容器时，是可变（元素的个数、值和顺序可变）并且有序（列表一旦确定，列表中元素的顺序就确定）的。
##2.列表中的元素
任何类型的数据都可以作为列表元素，而且同一个列表中元素的类型可以不一样。

#2. 获取列表
##1. 获取单个元素
###1. 语法
列表[下标] - 获取列表中指定下标对应的元素。
###2. 说明
列表 —> 结果是列表的表达式，例如：列表数据，存储列表的变量或者是结果是列表的函数调用表达式。
[] —> 固定写法。
下标（也称索引） —> 列表中每一个元素都有一个下标来表示该元素在列表中的位置。
下标的范围有两种：
0 ~ (列表长度-1)（列表长度指的是列表中元素的个数）。
-1 ~ (-长度)
0表示第一个元素，-1表示最后一个元素，上面两种写法即表示顺序和逆序。
注：下标不能越界
##2. 获取部分元素（列表切片）
###1. 语法
列表[开始下标: 结束下标: 步长] —> 从开始下标开始取，每次下标值增加步长去取下一个元素，直到取到结束下标前一个元素为止。
返回的数据是列表（取不到则返回空列表）。
###2. 说明
开始下标: 结束下标: 步长 —> 步长是针对下标位置来说的，不要太关注下标本身的数值，因为相同位置的数值可以是正数也可以是负数。
例[-1, 2, -1]是可以取到值的。(前提是list长度大于等于4)
###3. 注意
步长为正时，开始下标元素的位置要在结束下标元素位置的前面，否则返回结果为空。（从前往后取）
步长为负时，开始下标元素的位置要在结束下标元素位置的后面，否则返回结果为空。（从后往前取）
##3. 切片语法的变形
###1. 省略步长
列表[开始下标: 结束下标] — 步长为1
###2. 省略开始下标
列表[: 结束下标: 步长] 或 列表[: 结束下标]
对前一种情况来说：
步长为正从下标为0开始（从头开始）
步长为负从下标为-1开始（从尾开始）

列表[开始下标::步长] 或 列表[开始下标:]
